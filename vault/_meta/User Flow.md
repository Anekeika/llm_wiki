# User Flow — Сценарии использования

## Назначение

Wiki — инструмент работы Claude, не учебный проект. Каждый проект получает свой раздел в `wiki/`. Claude накапливает контекст, архитектурные решения, паттерны и на старте каждой сессии уже «в теме».

## Роли в системе

| Роль           | Кто   | Что делает                                                    |
|----------------|-------|---------------------------------------------------------------|
| **Куратор**    | Ты    | Указывает что важно, направляет, проверяет результат          |
| **Компилятор** | Claude | Пишет и поддерживает wiki, кросс-ссылки, синтез, решения     |

---

## Флоу 0 — Старт сессии по проекту

```
Открываешь Claude Code для работы над проектом
        ↓
Claude читает wiki/<project>/index.md
        ↓
Claude сразу в контексте:
архитектура, последние решения, открытые вопросы
        ↓
Работаете — без пересказа с нуля
```

---

## Флоу 1 — Добавление источника в wiki проекта

```
Завершили важный блок работы / приняли решение
        ↓
Говоришь Claude: "зафиксируй в wiki/pravdaserm/"
        ↓
Claude обсуждает с тобой что именно важно
        ↓
Ты указываешь акценты
        ↓
Claude пишет / обновляет страницы в wiki/<project>/
        ↓
Обновляет index.md и log.md
        ↓
Проверяешь в Obsidian
```

**Время:** 5–10 минут после завершения значимого блока работы.

## Флоу 2 — Исследовательский запрос

```
У тебя вопрос по теме
        ↓
Спрашиваешь LLM напрямую
        ↓
LLM читает index.md → находит страницы
        ↓
LLM синтезирует ответ
        ↓
Выбираешь формат вывода:
  - Markdown-страница
  - Таблица сравнения
  - Marp-слайды
  - График (matplotlib)
        ↓
(Опционально) "файлуй ответ в wiki/"
        ↓
Ответ становится частью базы знаний
```

## Флоу 3 — Сессия работы (типичный день)

```
Открываешь Obsidian + Claude Code
        ↓
LLM читает index.md + log.md
(понимает где остановились)
        ↓
[Если есть новые источники] → Ingest
        ↓
[Если есть вопросы] → Query
        ↓
[Раз в неделю] → Lint
        ↓
Просматриваешь граф Obsidian
(видишь как растёт вики)
```

## Флоу 4 — Lint-сессия (раз в неделю)

```
Говоришь: "проверь здоровье вики"
        ↓
LLM проходится по всем страницам
        ↓
Выдаёт отчёт:
  - Противоречия: [список]
  - Устаревшее: [список]
  - Orphan-страницы: [список]
  - Нет страницы но упоминается: [список]
  - Новые вопросы для исследования: [список]
        ↓
Ты решаешь что фиксить сейчас
        ↓
LLM вносит правки
```

## Флоу 5 — Авто-захват сессии (claude-memory-compiler)

```
Ты работаешь с Claude Code как обычно
        ↓
Сессия заканчивается (или начинается compaction)
        ↓
Хук автоматически запускает flush.py
        ↓
Claude Agent SDK решает что важно сохранить:
решения, паттерны, уроки, gotchas
        ↓
Запись добавляется в daily/YYYY-MM-DD.md
        ↓
После 18:00 (или вручную) — compile.py
        ↓
wiki/concepts/ и wiki/qa/ обновляются
        ↓
На следующей сессии — SessionStart hook
инжектит index.md в контекст Claude
```

**Ты ничего не делаешь** — пайплайн работает сам. Опционально: запустить `query.py` или `lint.py` вручную.

## UX-стек

| Задача          | Инструмент                                            |
|-----------------|-------------------------------------------------------|
| Просмотр вики   | Obsidian (граф, бэклинки, Dataview)                   |
| Клиппинг статей | Obsidian Web Clipper                                  |
| Общение с LLM   | Claude Code                                           |
| Слайды из вики  | Marp (плагин Obsidian)                                |
| Поиск по вики   | index.md → LLM (на старте) / query.py / qmd (после ~200 страниц) |
| Версионирование | Git (вики = git repo)                                 |
