# Pipeline — Полный пайплайн LLM Wiki

## Назначение

Персистентная база знаний для улучшения работы Claude по проектам. Claude ведёт отдельную wiki на каждый проект —
решения, паттерны, контекст накапливаются между сессиями.

```
Проект A  →  wiki/project-a/
Проект B  →  wiki/project-b/
...
```

В начале сессии Claude читает `index.md` нужного проекта и сразу в контексте — без пересказа с нуля.

---

## Обзор

```
Источник → raw/ → Ingest → wiki/<project>/ → Query → Output → (→ wiki/)
                                                ↑
                                             Lint
```

Три слоя:

- **raw/** — исходники, иммутабельные (статьи, PDF, клиппинги, транскрипты)
- **wiki/** — LLM-генерированные Markdown-файлы, живая база знаний по проектам
- **schema** — `CLAUDE.md` / `AGENTS.md`, конфиг поведения LLM

---

## Два режима пайплайна

| Режим      | Источник                              | Запуск              |
|------------|---------------------------------------|---------------------|
| **Ручной** | Статьи, PDF, клиппинги из Web Clipper | Ты инициируешь      |
| **Авто**   | Разговоры с Claude Code               | Хуки, автоматически |

Авто-режим реализуется через **claude-memory-compiler** — см. ниже.

---

## Операция 1 — Ingest (добавление источника)

```
1. Кладёшь файл в raw/
2. Говоришь LLM: «обработай этот источник»
3. LLM читает файл
4. LLM пишет summary-страницу в wiki/sources/
5. LLM обновляет затронутые entity- и concept-страницы (5–15 файлов)
6. LLM обновляет index.md
7. LLM добавляет запись в log.md
```

**Инструменты сбора источников:**

- Obsidian Web Clipper — конвертация веб-статей в Markdown
- Скачивание изображений локально (Ctrl+Shift+D в Obsidian)
- Вручную: PDF, заметки, транскрипты

---

## Операция 2 — Query (запрос к вики)

```
1. Задаёшь вопрос LLM
2. LLM читает index.md → находит релевантные страницы
3. LLM читает страницы → синтезирует ответ
4. Ответ в выбранном формате: .md, таблица, Marp-слайды, matplotlib
5. (Опционально) файлуешь ответ обратно в wiki/ → он компаундится
```

**Ключевой момент:** вопросы и ответы накапливаются в вики — исследование не исчезает в истории чата.

---

## Операция 3 — Lint (здоровье вики)

```
Периодически:
1. LLM проверяет противоречия между страницами
2. Находит устаревшие утверждения (новые источники опровергли старые)
3. Находит orphan-страницы (нет входящих ссылок)
4. Находит «упомянуто, но нет страницы» → предлагает создать
5. Предлагает новые вопросы для исследования
```

## Авто-пайплайн — claude-memory-compiler

Адаптация метода Карпати для захвата разговоров с Claude Code.

```
Разговор с Claude Code
        ↓
SessionEnd / PreCompact hook (автоматически)
        ↓
flush.py → Claude Agent SDK извлекает решения и уроки
        ↓
daily/YYYY-MM-DD.md  (append-only лог за день)
        ↓
compile.py (после 18:00 или вручную)
        ↓
wiki/concepts/, wiki/connections/, wiki/qa/
        ↓
SessionStart hook → инжектит index.md в следующую сессию
        ↓
Цикл повторяется
```

**Ключевые скрипты** (живут в `E:\anekeika-llm-wiki\scripts\`):

| Скрипт     | Команда                                   | Что делает                     |
|------------|-------------------------------------------|--------------------------------|
| flush.py   | авто (хук)                                | захватывает сессию → daily log |
| compile.py | `uv run python scripts/compile.py`        | daily logs → wiki-статьи       |
| query.py   | `uv run python scripts/query.py "вопрос"` | Q&A по вики без RAG            |
| lint.py    | `uv run python scripts/lint.py`           | 7 проверок здоровья вики       |

**Почему без RAG:** на личном масштабе (50–500 статей) LLM читает `index.md` и находит нужное лучше, чем cosine
similarity. RAG нужен при ~2000+ страниц.

---

## Специальные файлы

### index.md

Каталог всего содержимого. LLM читает его первым при каждом запросе. Структура:

```
## Concepts
- [[токенизация]] — BPE, WordPiece, SentencePiece. Источники: 3
- [[attention]] — механизм внимания, варианты. Источники: 5

## Sources
- [[статья-название]] — краткое описание, дата
```

### log.md

Append-only журнал. Каждая запись:

```
## [2026-04-18] ingest | Название источника
Краткое описание что было добавлено/обновлено.
```

### CLAUDE.md (schema)

Главный конфиг — определяет структуру вики, конвенции именования, воркфлоу. Эволюционирует вместе с проектом.

### daily/YYYY-MM-DD.md

Append-only суточные логи от claude-memory-compiler. Сырой материал до compile.py.
