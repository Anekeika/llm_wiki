# Предложения — Как развивать систему

## 1. Первый шаг — wiki/pravdaserm/

Pravdaserm уже в активной работе. Самое ценное — зафиксировать текущий контекст пока он свеж.

За одну сессию скомпилировать:
- Стек и архитектура (FSD, Nuxt 4, Pinia, HTTP-клиент)
- Auth flow и query store — как работает сейчас
- Ключевые решения последних PR (admin sidebar, query status patch, error handling)
- Открытые вопросы и известные точки боли

**Результат:** каждая следующая сессия по Pravdaserm начинается с контекстом, не с нуля.

---

## 2. Поднять claude-memory-compiler

Репо [coleam00/claude-memory-compiler](https://github.com/coleam00/claude-memory-compiler) — авто-захват сессий.

**Что даёт:**
- Каждый разговор с Claude автоматически попадает в daily log
- `compile.py` превращает логи в структурированные wiki-статьи
- `SessionStart` хук инжектит index.md в начало следующей сессии
- `query.py` и `lint.py` — поиск и здоровье вики без RAG

**Как поднять** (скажи — сделаю):
```bash
git clone https://github.com/coleam00/claude-memory-compiler E:\anekeika-llm-wiki\memory-compiler
cd E:\anekeika-llm-wiki\memory-compiler && uv sync
# затем — скопировать хуки в .claude/settings.json каждого проекта
```

**Где живёт:** `E:\anekeika-llm-wiki\memory-compiler\` — вне Obsidian vault.

---

## 3. Схема (CLAUDE.md в vault)

Создать `CLAUDE.md` прямо в Obsidian vault — общие конвенции для Claude:

```
Конвенции именования файлов
Структура frontmatter
Что фиксировать обязательно: решения, паттерны, gotchas
Что НЕ фиксировать: временные баги, тривиальные правки
Как ссылаться между wiki разных проектов
```

Плюс локальный `CLAUDE.md` в каждом `wiki/<project>/` — специфика проекта.

---

## 4. Конвенция коммитов

Wiki уже в git. Читаемый `git log` = история работы:

```
wiki(pravdaserm): ingest auth flow decision
wiki(pravdaserm): lint weekly
wiki: add project-2 index
schema: update naming conventions
```

---

## 5. Сверка wiki с кодом при старте сессии

Предлагаю добавить в воркфлоу старта: Claude сверяет 2–3 ключевых точки из index.md с текущим кодом.

Пример: «в wiki написано что HTTP-клиент делает 2 retry — проверяю `shared/api/http.ts`».

Это выловит устаревшие записи до того как они введут в заблуждение.

---

## Приоритет действий

1. **Сейчас:** создать `wiki/pravdaserm/` — зафиксировать текущий контекст проекта
2. **Создать** `CLAUDE.md` в vault — общие конвенции
3. **Поднять** claude-memory-compiler — авто-захват всех сессий
4. **По мере роста:** добавлять новые проекты в `wiki/<project>/`
5. **При ~200 страницах:** подключить qmd или свой поисковик
