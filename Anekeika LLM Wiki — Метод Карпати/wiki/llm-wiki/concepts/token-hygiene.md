# Оптимизация токенов Claude Code

**TLDR:** 98.5% токенов уходит на перечитывание истории, а не на работу. Контекстная гигиена важнее тарифа.

## Механика токенов

- Каждое сообщение — Claude перечитывает всю историю с нуля
- Стартовый контекст: ~51k токенов до первого сообщения (системные промпты, инструменты)
- Кэш живёт ровно 5 минут — перерыв дольше = полный пересчёт
- Автокомпакт срабатывает на 95% — к этому моменту середина контекста уже деградирует (Loss in the Middle)
- Подписка Max $200/мес = ~$3000-5000 в API-эквиваленте

## Базовые хаки (1–9)

| # | Хак | Экономия |
|---|---|---|
| 1 | `/clear` между задачами | до 140k токенов за итерацию |
| 2 | Отключить ненужные MCP серверы | до 18k токенов/сервер за сообщение |
| 3 | Объединять промты в одно сообщение | 60–70% за счёт устранения циклов |
| 4 | Plan Mode перед сложной задачей (95% уверенности) | предотвращает дорогие откаты |
| 5 | `/context` и `/cost` — аудит что жрёт токены | диагностика |
| 6 | Строка статуса `/status line` | мониторинг заполнения |
| 7 | Дашборд usage (остаток + время до сброса) | контроль лимитов |
| 8 | Давать только нужный контекст (функцию, не весь repo) | до 96% экономии |
| 9 | Следить и останавливать Claude если пошёл не туда | предотвращает циклы |

## Продвинутые хаки (10–14)

**10. CLAUDE.md как индекс, не библиотека**
- Максимум 200 строк
- Только правила, стек, команды сборки, правило 95%
- Ссылки на `Architecture.md`, `Knowledge.md` — не сам контент
- Писать на английском (кириллица токенизируется в 1.5–2x дороже)

**11. Точные ссылки через @filename**
- Не "найди в репозитории", а "@src/utils/auth.ts строка 42"
- Предотвращает сканирование всего проекта (2k токенов vs 30k+)

**12. `/compact` вручную на 60%**
- Не ждать автокомпакта на 95% — к тому моменту середина уже плывёт
- Явно указывать что сохранить: архитектурные решения, текущая задача, структура файлов

**13. 5-минутное окно кэша**
- Кэш живёт 5 минут
- Уход на 6 минут → следующее сообщение пересчитывается с нуля
- Перед паузой: `/compact` или `/clear`

**14. Лимитировать вывод shell-команд**
- `git log` без лимита = 200 коммитов в контекст
- `npm install` = сотни строк логов
- Прописать в CLAUDE.md: лимит для всех команд

## Экспертные хаки (15–18)

**15. Иерархия моделей**
- Sonnet: 80% задач (основной инструмент)
- Haiku: 15% (субагенты, форматирование, простые задачи)
- Opus: 5% (глубокая архитектура, критическое планирование)

**16. Субагенты стоят в 7–10x дороже**
- Каждый субагент просыпается со своим полным контекстом
- Делегировать им только одноразовые задачи с моделью Haiku

**17. Пиковые часы: 15:00–21:00 МСК**
- Тот же лимит расходуется быстрее
- Большие рефакторги и сложные задачи — на вечер/ночь/выходные

**18. CLAUDE.md как "живая конституция"**
- Секция "что не объяснять дважды"
- Одно правило = одна строка, максимум 15 слов
- Читается автоматически при каждой сессии = бесплатный контекст

## Бонусные хаки (19–24)

**19. `.claudeignore`**
- Как `.gitignore` — перечислить что Claude не должен читать
- `node_modules/`, `dist/`, `*.log`, файлы сборки

**20. Иерархия CLAUDE.md**
- Глобальный: личный стиль кода, одинаково везде
- Проектный: архитектура проекта
- Директорий: правила конкретного модуля
- Без дублирования между уровнями

**21. Extended Thinking: 10k вместо 32k**
- По умолчанию 31999 токенов на "обдумывание" каждого запроса
- Для 90% задач достаточно 10k — экономия 70% thinking-токенов
- Настраивается через API (конфиг запроса)

**22. memory.md в корне проекта**
- Claude забывает всё между сессиями
- `memory.md` с: архитектура, паттерны которые работают, паттерны которые не работают, пути к ключевым файлам
- Новая сессия начинается с чтения, не с объяснений

**23. N8N для рутинных задач**
- Claude даёт команду через webhook → N8N выполняет → возвращает результат
- Токены не капают пока Claude ждёт внешнее API
- Пример: анализ YouTube (скачать + транскрибировать + анализировать) через N8N

**24. Caveman Output**
В CLAUDE.md добавить:
```
output style: No preamble. Tool result first. No explanation of action. Stop.
```
- Убирает вежливые вступления
- Экономия 50–75% исходящих токенов на коммуникационных сообщениях
- Работает только на исходящих, не на коде/анализе
