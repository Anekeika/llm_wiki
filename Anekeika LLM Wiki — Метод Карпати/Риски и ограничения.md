# Риски и ограничения

## Технические ограничения

### Масштаб wiki
- **Index-based retrieval работает до ~200–500 страниц на проект.** После этого index.md не влезает в контекст — нужен поисковик (qmd или самодельный).
- При многих проектах одновременно — контекст растёт быстрее: нужно читать только index.md нужного проекта, не всех.
- **~2000+ страниц** — нужен настоящий RAG с эмбеддингами.

### Контекстное окно
- Старт сессии с чтением index.md занимает токены до начала работы.
- Lint по всей wiki проекта дорог — лучше делать по одному проекту за раз.
- Один Ingest трогает 5–15 файлов — нужен достаточный контекст.

### Качество компиляции
- Claude может неточно синтезировать → нужна вычитка важных решений.
- Качество вики напрямую зависит от качества `CLAUDE.md` (схемы).
- Без явного указания «зафиксируй это» — решения теряются в истории чата.

## Процессные риски

### Нет привычки фиксировать
- Главный риск: вики не растёт, потому что забываешь говорить «зафиксируй».
- Митигация: claude-memory-compiler захватывает сессии автоматически.

### Дрейф схемы между проектами
- Разные проекты могут получить несовместимые конвенции именования.
- Правило: один `CLAUDE.md` на vault с общими конвенциями + локальный `CLAUDE.md` в каждом `wiki/<project>/` для специфики.

### Устаревание
- Wiki фиксирует момент времени. Архитектурное решение могло измениться — а в wiki старая версия.
- Митигация: при старте сессии Claude сверяет index.md с текущим кодом по ключевым точкам.

### Orphan-страницы
- Страницы без входящих ссылок «теряются» — не попадают в граф, не используются при Query.
- Решение: регулярный Lint.

### Зависимость от Claude
- Весь контент пишет Claude — без доступа wiki «замораживается».
- Митигация: Git-история сохраняет все версии, Markdown читается без инструментов.

## Стоимость

| Операция | Токены | Частота |
|----------|--------|---------|
| Старт сессии (чтение index.md) | низко | каждая сессия |
| Ingest решения/блока работы | средне | по завершении блока |
| Query по wiki проекта | средне | по необходимости |
| Lint проекта | высоко | раз в неделю |
| claude-memory-compiler flush | авто | конец каждой сессии |

**Claude Max/Team покрывает Agent SDK без отдельного биллинга.**

---

## Организационные ограничения

- **Требует дисциплины.** Если не говоришь «зафиксируй» — ничего не накапливается (без memory-compiler).
- **Нет уведомлений.** Claude не скажет сам «это решение противоречит wiki» — только если спросишь.
- **Один Claude — один проект за раз.** Параллельная работа по нескольким wiki не поддерживается нативно.

---

## Что НЕ решает этот подход

- ❌ Автоматическую синхронизацию с кодом (wiki может устареть)
- ❌ Реалтайм-алерты на противоречия
- ❌ Совместную работу нескольких разработчиков над одной wiki
- ❌ Поиск без LLM (на больших объёмах нужен отдельный инструмент)
